单例模式：

回顾一下Unity生命周期：(76条消息) Unity简单机制深究之一——谁调的Awake？_Don里个冬的博客-CSDN博客

在游戏开发中，一般各种manager（继承自 MonoBehaviour 的类，实例由Unity 游戏引擎创建，不能通过构造函数创建）在场景中只需要一个实例，并且它的数据需要在别的类中被访问（比如后面提到的使用它的内部委托对象进行订阅），那么它就很适合作为一个单例进行使用。

  1. 通常来说，在这个类中声明一个这个类的public static对象即可（保证可以直接通过类访问，而不需要通过实例化这个类），一般命名为Instance，可以被其他类访问。
  2. 在Awake的时候，要将这个Instance第一时间赋值为this对象（Awake作为MonoBehaviour的第一个生命周期，是在MonoBehaviour实例化的时候调用）；一般同时还要使用 DontDestroyOnLoad() 方法让 manager 在切换游戏场景的时候不会被销毁。
  3. 编译时：先生成Instance static对象（因为一个类只有一个这个static对象，所以不会存在这个static对象的时候再生成一个static对象，无限递归的情况，但是说明这个Instance的Instance对象就是它自己…），然后Unity生成一个manager实例对象，然后调用Awake，在其中将指向这个实例对象的this赋值给Instance。
  4. 为了进行数据的访问控制（不能随便读写的数据，通常设置为private static，用一个public static相同数据类型的属性作为访问的接口），更常见的做法是使用get/set实现私有数据读写：C#——get方法和set方法（属性） - 简书 (jianshu.com)：

C#提供一个使用属性的基本模式，该模式利用get访问器返回私有字段的值，利用set访问器对私有字段先验证再赋值。

private static Class_ class;

public static Class_ Class {get {return class;}}

或者

public Class_ Class {get => class;}

如果只有get，那就是只读属性；如果只有set，那就是只写属性。

更通俗的理解：

一个单例中：

  1. 需要一个private的自己属性(this)以及一个public替身属性；
  2. 这个替身代替真正的单例在别的类中被访问，从而实现对真正的单例的读写控制。

委托和事件：

  1. 委托可以用于实现观察者模式。
    1. 委托有很好的封装性。
    2. 委托的实例化与它的执行是可以在不同的对象中完成的。
  2. Action是一个c#内部声明的泛型委托，模板T是传入的参数类型，无返回值。
  3. 所有委托类都可以作为一个event(事件)被实例化，只要在前面加上event关键字即可。

比如当游戏中点击鼠标使得player发生移动的时候，如果使用编辑器event面板进行管理，那么后期如果需要修改物体那么就需要重新拖动物体，设置函数，非常费时费力。

如果使用委托，那么就可以直接在player（或者其他需要响应鼠标点击的物体）对应的script中将函数订阅到鼠标类的委托/事件中，当鼠标点击时，需要调用委托/事件，委托/事件就会依次执行订阅了该委托/事件的函数（类似回调来响应这个点击操作），这样可以将管理委托/事件以及订阅委托/事件的类解耦，彼此不需要知道具体的实现。

如果是使用类的多态，那么就产生了层级关系，多个类必须继承自同一个父类，才可以用父类指针调用子类override的同名同参同返回值函数。而订阅委托/事件的类只要保证添加到委托/事件内的函数的参数和返回值和委托/事件声明时的保持一致就可以了，可以使用的范围更广。

  1. 事件是一个封装过的委托，它不能被直接赋值，但是可以添加或取消订阅（+= and -=）。

    1. 保护委托字段，对外不开放，所以外部对象没法直接操作委托。提供了Add和Remove方法，供外部对象订阅事件和取消事件。

    2. 事件的处理方法在对象外部定义，而事件的执行是在对象的内部，至于事件的触发，何时何地无所谓。

更通俗的理解：

实例委托是一个用于管理与声明委托相同参数/返回值类型的函数的一个实例对象，这个对象可以被其他类函数订阅，订阅后相当于就是加入了这个函数管理列表中，当需要使用这个委托的时候（比如鼠标点击），就会依次调用内部的函数，执行各个类指定的，需要执行的内容。

大致使用规则：

假如已经声明一个void Delegate(type1 var1, type2 var2);

并且实例化了一个Delegate delegate;

委托的赋值：

  1. 可以当做参数传递，直接将实参（外部函数）隐式转换成形参（委托）的类型；

  2. delegate += function;

  3. delegate = new Delegate(function);

或者

delegate = function;// 隐式转换

委托的调用：delegate?.invoke(var1, var2);

?.invoke()：

若event不为null，则invoke，这是C#6的新语法。 ?.称为空值传播运算符。

委托中，匿名函数的用法

方法一：

匿名方法表达式的语法包含如下组成部分。

  1. delegate类型关键字。

  2. 参数列表，如果语句块没有任何参数则可以省略。

  3. 语句块，它包含了匿名方法的代码。

    delegate ( 参数列表 ) { 代码块 }

方法二：

lambda表达式：

// var作为匿名函数的传入参数

delegate += var => function内容;

// 若没有参数：

delegate += () => function内容;
