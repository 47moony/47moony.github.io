static（全局/ 局部/ 成员）

基本定义：

C++ static静态成员函数详解 (biancheng.net)

(76条消息) C++程序员应了解的那些事（55）静态成员函数没有this指针，不与类的实例（对象）“挂钩”【即不能声明为const、volatile、virtual】_so~what的博客-CSDN博客_为什么静态成员函数没有this指针

(76条消息) c++调用静态函数的方法_ComputerInBook的博客-CSDN博客_c++调用静态方法

  1. static函数只能访问static成员变量；
  2. static成员变量会被所有对象，包括派生类对象共享；
  3. static成员变量只能在类外进行初始化（除了静态常量成员可以在类内初始化），类外定义不用加static；
  4. 普通成员函数可以访问所有成员，它有this指针，编译器会在成员函数的参数列表第一位加上Class this；
  5. 静态成员函数没有this指针，不与类的实例（对象）“挂钩”，所以C++静态成员函数不能声明为const、volatile、virtual。
  6. 可以通过类名访问static函数/成员。（和python的@staticmethod用法一致）新标准的c++可以通过实例化对象来调用静态成员函数。
  7. static有“延长生命周期”和“默认初始化为0”两个特殊性质。

static & extern：

extern就是声明，不会分配内存空间；而定义则会分配内存空间。

(85条消息) C++静态全局变量的不可见性_林多的博客-CSDN博客

  1. 全局变量本身具有可见性，可以在初其定义的文件外，通过extern关键字声明该全局变量，但是不能定义，否则就是重定义。
  2. 如果这个全局变量是static修饰的，那么这种可见性不成立，即不可以通过extern来使用。可以解决重命名的问题，因为这样可以保证这个全局变量只在声明的文件中使用，别的文件不可使用。（C++不鼓励使用static来实现数据隐藏）
  3. const全局变量不能在外部被访问，除非声明该变量时加了extern。所以按理来说也是有“隐藏”的作用。
  4. 按全局变量和静态全局（外部）变量，都是采用静态存储方式，并非不加static的全局变量就是动态存储的。全局变量和静态全局变量，只是作用范围不同（静态全局变量只限于本文件引用），但都是在编译时分配内存。

extern & include:

  1. 只希望使用某个头文件中的全局变量或全局方法，用extern更好，因为include相当于是头文件中的内容搬到了#include的位置，如果是需要在2个及以上的文件内访问同一个定义在头文件内的全局变量（头文件内的定义不算）时，使用include会导致全局变量被重复定义，编译无法通过。
  2. 所以，如果一定要在多个文件中用include，其实相当于就放弃了全局变量的可见性和共享性，为了避免重复定义，应该在头文件中将全局变量定义为static，这样的话每个对应的文件中该全局变量都是static的，不会造成重定义。
  3. 当然反过来说，如果一定不会被多个文件include，那么这个头文件中的全局变量就可以不用加static。
  4. 一般都不在头文件里面定义全局变量，但是可以用extern声明（？）。(85条消息) static与全局变量_赵同学的博客-CSDN博客_static全局变量

static局部变量（再整理）：

C语言局部变量、静态局部变量、全局变量与静态全局变量！ - 知乎 (zhihu.com)

  1. 静态局部变量和普通局部变量不同。静态局部变量也是定义在函数内部的，静态局部变量定义时前面要加static关键字来标识，静态局部变量所在的函数在多调用多次时，只有第一次才经历变量定义和初始化，以后多次在调用时不再定义和初始化，而是维持之前上一次调用时执行后这个变量的值。本次接着来使用。
  2. 静态局部变量在第一次函数被调用时创造并初始化，但在函数退出时它不死亡，而是保持其值等待函数下一次被调用。下次调用时不再重新创造和初始化该变量，而是直接用上一次留下的值为基础来进行操作。
  3. 静态局部变量的这种特性，和全局变量非常类似。它们的相同点是都创造和初始化一次，以后调用时值保持上次的不变。不同点在于作用域不同。
  4. 静态局部变量属于静态存储类别，在静态存储区分配存储单元，在整个运行期间都不释放。
  5. 而自动变量(动态局部变量)属于动态存储类别，占动态存储空间，函数调用后释放
  6. 静态局部变量在编译时赋初值，程序运次时已经有初值，以后每次调用函数不再重新赋初值而只是保留上次函数调用结束时的值。
  7. 静态局部变量编译时赋初值默认是0
  8. 而自动变量，如果不赋初值则它的值是不确定的值。
  9. 静态局部变量在函数调用结束后仍然存在，但其他函数不能引用它。
  10. 如果用static修饰全局变量，那么该全局变量不能被其他文件引用。不能再被extern（extern和static都是修饰符）
  11. 如果用static修饰函数，那么该函数本文件内可以引用而其他文件不能引用。

  2. 修饰指针：const char* p ：表示指向的内容不能被改变；
    char* const p ： 表示指向的地址不能改变，内容可以改变。
  3. 修饰函数参数：
    1. 防止传入的参数代表的内容在函数体内被改变，但仅对指针和引用有意义，代表在函数体内不能修改该指针所指的内容，起到保护作用。

  4. 修饰返回值：也是用const来修饰返回的指针或引用，保护指针指向的内容或引用的内容不被修改，也常用于运算符重载。归根究底就是使得函数调用表达式不能作为左值。
  5. 修饰成员函数：eg. int get_score() const ([const 修饰成员函数 前后用法（effective c++ 03) - lightmare - 博客园 (cnblogs.com)](https://www.cnblogs.com/lightmare/p/10398824.html#:~:text=const修饰成员函数的两个作用. 在设计类的时候，一个原则就是对于不改变数据成员的成员函数都要在后面加 const，而对于改变数据成员的成员函数不能加 const。. 所以 const 关键字对成员函数的行为作了更加明确的限定 ：有,const 还有什么好处呢？. “ 获得能力：可以操作常量对象 ”，其实应该是常量（即 const）对象可以调用 const 成员函数，而不能调用非const修饰的函数。.))
    1. 只能读成员变量，不能写；
    2. 只能在函数中调用const成员函数，不能调用非const成员函数；为什么不能调用非const函数？因为非const函数可能修改数据成员，const成员函数是不能修改数据成员的，所以在const成员函数内只能调用const函数。
    3. const类对象只能调用const成员函数，不能调用非const成员函数。
  6. 修饰成员变量：不能在类定义处初始化，只能通过构造函数初始化列表初始化，必须有构造函数。

vs. define宏定义

编译器处理方式不同
define –> 在预处理阶段进行替换
const –> 在编译阶段确定其值

类型检查
define –> 无类型，不进行类型安全检查，可能会产生错误
const –> 有数据类型，编译时会检查

内存空间
define –> 不分配内存，给出的是立即数，用了多少次就进行多少次替换，在内存中会有多个拷贝，消耗内存大
const –> 在静态存储区分配空间，程序运行中在内存中只有一次拷贝

在编译时，编译器通常不为const分配空间而是保存在符号表中，使得成为编译期间的常量，没有存储与读内存的操作，效率高。
宏替换只做替换，不做表达式计算。

内存分配：

1.代码区：存放CPU指令码。

2.常量区：存放只读常量，该区只读，不可写。

3.静态区：存放静态变量。该区在程序编译完成后就决定了其大小，程序运行期间该区的大小不会变。该区可读写。

4.动态区：又分为堆区和栈区，程序运行期间其大小处于动态变化中。处于该区的变量也会时而被创建时而被销毁。

指针作为参数进行传递

指针的内容，存的是它指向对象所在的内存地址。

*指针，就是取它指向对象所在的内存中的数据。

因此，指针本身的存储也需要地址，因此如果对指针取地址，那么就能得到指针的地址。

传递值还是传递地址？

一般指针作为形参，外面的实参都是一个需要指针指向的对象，那么参数列表编译时就是int* p = &a；*p是形参，a是实参，将实参a取得的地址传递给 *p，从而可以对指向的地址内存中的内容进行修改： *p = val，外部的a的值就会变成val。

但也存在一种用法，是传递指向指针的指针，也就是指针的地址，将指针的地址作为值来传递，从而对指针的内容进行修改，也即修改指针的指向。对指针取地址，&(*p)：(76条消息) 指针地址作为参数传递_jackeven的博客-CSDN博客_指针地址参数传递

指针的引用也可以用于修改外部传入的指针所指向的对象，也就是修改指针的内容：#图解 轻松看懂「指针的引用*&」 - 知乎 (zhihu.com)

总结

如果传入的参数类型是值，那么函数调用的时候就可以传入一个右值，如果在函数调用之后不会再用到这个值，那么给它命名也没有意义；

同理，如果返回的数据类型是一个值，那么返回值就可以是一个右值，因为之后不会再用到这个值了，它只存在于这个函数的作用域，给它命名也没有意义；

如果传入的参数是指针或者引用，那么函数调用的时候传入的必须是一个左值，一定不能是右值，因为右值在赋值代码结束后就会被系统回收；

同理，如果返回的数据类型是一个指针或者引用，那么返回值必须是一个指针（比如内部new在堆上的对象指针）或者是一个函数外部变量的引用，一定不能是右值，因为右值在赋值代码结束后就会被系统回收。
