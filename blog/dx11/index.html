参考计算出了Rroll * Rpitch * RYaw，然后分别乘forward[0, 0, 1, 0]以及up[0, 1, 0, 0]，得到旋转之后的forward和up，然后叉乘计算出right。

但是目前存在一个问题，有些角度之后移动pitch和yaw会反向。

总结：

目前讨论的所有变换都是在世界坐标系下展开的。

世界坐标系是一个以(0, 0, 0)为原点，一组标准正交基{(1, 0, 0), (0, 1, 0), (0, 0, 1)}为轴构成的。

旋转矩阵的推导：

  1. 首先选择特殊的点A(1, 0, 0)，B(0, 1, 0)，C(0, 0, 1)在旋转相应角度之后的坐标A’，B’，C’，然后通过这2个变换前后的坐标逆向推导出旋转矩阵R。
  2. 逆向推导过程中可以发现，旋转之后的点坐标A’，B’，C’，其实就 == 旋转矩阵R中按列排序的部分：第一列是A’，第二列是B’，第三列是C’。又因为，坐标系的三个轴向量的尾点就是A，B，C，因此，变换之后的A’，B’，C’就是旋转后的三个轴的尾点，即新的轴在世界坐标系下的表示。
  3. 因此，旋转矩阵R == 新坐标轴在世界坐标系下的表示，按列排序而成的矩阵M。
  4. 结合从一个空间变换到另一个空间的过渡矩阵M（Shader精要有推导；只考虑旋转），就可以发现，这个M就是旋转矩阵R。

$$ 世界坐标轴x,y,z分别={(1, 0, 0)^T, (0, 1, 0)^T, (0, 0, 1)^T}

∵[?]_{3*3}x=x’
∴[?]的第一列=x’
以此类推[?]的第二、三列，得到R

R世界坐标轴x,y,z=新的轴在世界坐标系下的表示x’,y’,z’
R=[x’,y’,z’按列排序] $$

Yaw/ Pitch/ Roll三个角是可以直接累加的，可以这么理解：

在进行了旋转之后，移动不会改变旋转的角度，假设我们已知摄像机的位置，那么就可以先移动回原点，之后再旋转摄像机坐标系轴到世界坐标系的位置。反过来，就是先进行旋转，然后进行移动，旋转各个轴只会增加各个角的角度，所以是直接累加的，并不需要做什么别的变换。
